# Performance Optimization

## 현재 적용된 최적화

### 1. 이미지 가상화 스크롤 (D019)
- **문제**: 수천~수만 개의 이미지를 한 번에 렌더링하면 브라우저가 버벅임
- **해결**: `@tanstack/react-virtual`로 뷰포트에 보이는 이미지만 렌더링
- **효과**: DOM 노드 수를 수십 개로 제한, 10만 개 이미지도 부드러운 스크롤
- **적용 범위**: VirtualImageGrid, VirtualImageList

### 2. 배치 업로드 제한 (D017)
- **문제**: 수백 개 파일 동시 업로드 시 서버/브라우저 메모리 폭발
- **해결**: 20개 파일씩 배치로 분할하여 순차 업로드
- **효과**: 메모리 사용량 제어, 서버 안정성 확보
- **UX**: 전체 진행률 표시로 사용자 체감 유지

### 3. 해시 기반 이미지 중복 제거
- **문제**: 같은 이미지를 여러 번 업로드하면 디스크 낭비
- **해결**: SHA-256 해시로 동일 파일 감지, 물리 파일은 한 번만 저장
- **효과**: 디스크 사용량 절감, 업로드 속도 향상 (이미 존재하면 저장 건너뜀)

### 4. 스토리지 경로 샤딩
- **문제**: 단일 디렉토리에 수십만 파일 저장 시 파일시스템 성능 저하
- **해결**: 해시의 첫 4자리로 2단계 디렉토리 분산 (256 x 256 = 65,536 디렉토리)
- **효과**: 디렉토리당 파일 수 제한으로 파일시스템 성능 유지

### 5. 비동기 I/O (Backend)
- **문제**: 동기 I/O는 동시 요청 처리에 병목
- **해결**: asyncpg (DB), aiofiles (파일), httpx (HTTP) — 전부 비동기
- **효과**: 단일 프로세스에서 높은 동시성 처리

### 6. DB 인덱싱
- `images.file_hash`: 업로드 시 중복 검사 O(log n)
- `images.folder_path`: 폴더별 이미지 조회 최적화
- `images.dataset_id`: 데이터셋별 이미지 조회 (FK 인덱스)

## 향후 최적화 과제

### Frontend

#### 이미지 썸네일 생성
- **현재**: 원본 이미지를 그대로 로딩 (대역폭 낭비)
- **개선**: 서버에서 썸네일 생성 후 캐시, 그리드에서는 썸네일만 로딩
- **효과**: 네트워크 대역폭 절감, 로딩 속도 대폭 향상

#### 이미지 지연 로딩 (Lazy Loading)
- **현재**: 가상화로 DOM은 제한되지만, 이미지 URL은 즉시 요청
- **개선**: Intersection Observer로 뷰포트 진입 시에만 이미지 로딩
- **효과**: 초기 로딩 시간 단축

#### React Query / SWR 도입
- **현재**: useState + useEffect로 각 컴포넌트에서 직접 API 호출
- **개선**: React Query로 캐싱, 중복 요청 제거, 자동 재검증
- **효과**: 네트워크 요청 수 감소, 페이지 전환 시 즉시 표시

### Backend

#### 페이지네이션 표준화
- **현재**: 일부 엔드포인트만 페이지네이션 적용
- **개선**: 공통 커서 기반 페이지네이션 + 총 개수 캐싱
- **효과**: 대량 데이터 조회 시 응답 시간 일정

#### 이미지 전처리 파이프라인
- **현재**: 업로드 시 해시 계산 + 저장만
- **개선**: 업로드 시 메타데이터 추출 (EXIF), 썸네일 생성, 포맷 변환
- **효과**: 조회 시 추가 처리 불필요

#### 캐싱 레이어
- **현재**: Redis 설정은 있으나 캐싱 미활용
- **개선**: 자주 조회되는 데이터 (프로젝트 목록, 폴더 트리) Redis 캐싱
- **효과**: DB 부하 감소, 응답 시간 단축

### AI Worker

#### 학습 데이터 전처리
- **현재**: 미구현
- **개선**: 학습 시작 전 데이터 전처리 파이프라인 (리사이징, 증강)
- **효과**: 학습 중 I/O 대기 시간 감소

#### 모델 캐싱
- **현재**: 미구현
- **개선**: 자주 사용되는 사전학습 모델 로컬 캐싱
- **효과**: 학습 시작 시간 단축
