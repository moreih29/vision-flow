# UX Patterns

## 네비게이션 구조

```
로그인/회원가입
  └── 프로젝트 목록 (/)
        └── 프로젝트 상세 (/projects/:id)
              ├── [탭] Data Pool → 데이터셋 상세 (폴더/이미지)
              └── [탭] Subsets → Subset 상세 (/projects/:id/subsets/:subsetId)
```

### 인증 흐름
- 비인증 사용자: `/login` 또는 `/register`로 제한
- 인증 후: `ProtectedRoute` 컴포넌트가 인증 상태를 검사하여 접근 제어
- JWT 토큰 만료: 7일. API 응답 401 시 자동으로 로그인 페이지로 리다이렉트

## 이미지 관리 UX

### 뷰 모드 전환
- **그리드 뷰**: 썸네일 기반 이미지 카드 배열 (기본)
- **리스트 뷰**: 파일명, 크기, 날짜 등 메타데이터 중심 테이블 뷰
- **트리 뷰**: 폴더 계층 구조 탐색 (왼쪽 사이드바)
- 뷰 모드는 버튼으로 즉시 전환 가능

### 다중 선택 (Multi-Select)
- **Ctrl + 클릭**: 개별 이미지 토글 선택/해제
- **Shift + 클릭**: 범위 선택 (마지막 선택 ~ 현재 클릭 사이 모든 이미지)
- **선택 후 액션**: 배치 삭제, 배치 이동, Subset에 추가
- `useMultiSelect` 훅으로 구현

### 드래그 앤 드롭
- **파일 업로드**: 브라우저 영역에 파일/폴더 드래그하여 업로드
- **폴더 이동**: 이미지를 폴더 위로 드래그하여 이동
- **폴더 간 이동**: 트리 뷰에서 폴더를 다른 폴더로 드래그

### 컨텍스트 메뉴 (우클릭)
- **이미지**: 삭제, 이동, 이름 변경
- **폴더**: 하위 폴더 생성, 이름 변경, 이동, 삭제
- 브라우저 기본 컨텍스트 메뉴 대체

### 배치 업로드
- 최대 20개 파일씩 배치 분할 업로드 (D017)
- 업로드 진행률 표시
- 중복 파일 자동 건너뛰기 (해시 기반)
- 폴더 구조 유지하며 업로드 (폴더 드래그)

## 폴더 관리 UX

### 브레드크럼 네비게이션
```
프로젝트 > 데이터셋 > 폴더A > 하위폴더B
```
- 각 단계 클릭으로 해당 경로로 이동
- 현재 위치를 항상 시각적으로 표시

### 폴더 트리 뷰
- 왼쪽 사이드바에 전체 폴더 계층 표시
- 펼치기/접기 토글
- 폴더 클릭으로 해당 경로의 이미지 로드
- 우클릭 컨텍스트 메뉴 통합
- 드래그앤드롭 정리

### 빈 폴더 지원
- 이미지 없이도 폴더를 명시적으로 생성 가능
- `folder_meta` 테이블로 추적
- 폴더 삭제 시 하위 이미지/폴더 일괄 삭제

## Subset 관리 UX

### Subset 생성 흐름
1. "새 Subset" 버튼 → Dialog 열림
2. 이름, 설명, Task Type 선택
3. Task Type: Classification / Object Detection / Instance Segmentation / Pose Estimation
4. 생성 후 이미지 추가 단계로 이동

### 이미지 멤버십 관리
- **추가**: ImageSelectionModal에서 Data Pool 이미지 선택 → Subset에 추가
- **제거**: Subset 상세 페이지에서 이미지 선택 → 제거
- Data Pool의 이미지는 여러 Subset에 속할 수 있음

### 라벨 클래스 관리
- Subset 상세 페이지에서 클래스 CRUD
- 각 클래스에 이름 + 색상(HEX) 지정
- 같은 Subset 내 클래스 이름 중복 불가

## 대화형 UI 원칙

### 확인 다이얼로그 (D022)
- 삭제, 이동 등 위험/비가역 작업 전 반드시 확인 요청
- shadcn AlertDialog 사용 (브라우저 기본 confirm 사용 금지)
- `useConfirmDialog` 훅: `const confirmed = await confirm("삭제하시겠습니까?")`

### 로딩 상태
- 데이터 로딩 중 스피너 또는 스켈레톤 표시
- 빈 상태(데이터 없음) 시 안내 메시지 + 액션 버튼

### 에러 처리
- API 에러 시 Toast 알림 (sonner)
- 네트워크 에러와 비즈니스 에러 구분

## 가상화 스크롤 (D019)

- 수천~수만 개의 이미지를 로딩 없이 부드럽게 스크롤
- `@tanstack/react-virtual`로 뷰포트에 보이는 이미지만 렌더링
- 그리드 뷰: 행 단위 가상화
- 리스트 뷰: 행 단위 가상화
- 스크롤 성능과 메모리 사용량의 균형
