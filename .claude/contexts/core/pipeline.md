# Pipeline Orchestration

## 현재 상태: 설계 완료, 구현 미시작

React Flow 라이브러리는 Frontend에 설치되어 있으나,
파이프라인 에디터 UI는 아직 구현되지 않았다.

## 핵심 개념

파이프라인은 여러 AI 모델과 전/후처리 단계를 연결하여
복잡한 추론 워크플로를 시각적으로 구성하는 기능이다.

### 기존 서비스와의 차별점
- Roboflow, V7 Labs: 단일 모델 학습/추론만 지원
- Vision Flow: 다중 모델 파이프라인 + 피드백 루프

## 노드 기반 시각적 편집기

### 기술: React Flow
- 노드와 엣지(연결선)로 구성된 DAG(Directed Acyclic Graph)
- 드래그앤드롭으로 노드 배치
- 노드 간 연결선으로 데이터 흐름 정의
- 줌, 팬, 미니맵 등 편집 보조 기능

### 노드 유형 (설계)

| 노드 타입 | 역할 | 입력 | 출력 |
|----------|------|------|------|
| Input | 이미지/데이터 소스 | 없음 | 이미지 |
| Preprocessing | 이미지 전처리 | 이미지 | 처리된 이미지 |
| Model | AI 모델 추론 | 이미지 | 추론 결과 |
| Postprocessing | 결과 후처리 | 추론 결과 | 처리된 결과 |
| Output | 최종 결과 출력 | 결과 | 없음 |
| Branch | 조건 분기 | 결과 | 분기된 결과 |
| Merge | 결과 합류 | 여러 결과 | 통합 결과 |

### 파이프라인 예시

#### 예시 1: 2단계 감지
```
Input → Object Detection (전체 이미지)
         │
         ▼
       Crop (감지된 영역 잘라내기)
         │
         ▼
       Classification (잘린 이미지 분류)
         │
         ▼
       Output (최종 분류 결과)
```

#### 예시 2: 병렬 처리
```
Input ──┬── Model A (물체 감지) ──┐
        │                        ├── Merge → Output
        └── Model B (텍스트 감지) ──┘
```

#### 예시 3: 피드백 루프
```
Input → Model → Output
         │
         ▼
       피드백 (추론 결과 → 학습 데이터로)
         │
         ▼
       사람 검증 → 학습 데이터 편입
```

## 파이프라인 실행 아키텍처 (설계)

### 프론트엔드
- React Flow 기반 시각적 편집기
- 파이프라인 정의 (노드 + 엣지) → JSON으로 직렬화
- 실행 상태 시각화 (각 노드의 진행 상태 표시)

### 백엔드
- 파이프라인 정의 저장 (DB)
- 실행 요청 → DAG 토폴로지 순서로 노드 실행
- 각 노드의 출력을 다음 노드의 입력으로 전달
- 실행 상태 관리 (대기/실행 중/완료/실패)

### AI Worker
- 개별 모델 추론 실행 (파이프라인의 Model 노드)
- Celery 태스크로 비동기 처리
- 결과를 다음 노드로 전달

## 데이터 피드백 루프

### 개념
모델의 추론 결과를 학습 데이터로 재활용하여 지속적으로 모델을 개선하는 사이클.

### 흐름
```
1. 새 이미지에 대해 모델 추론
2. 추론 결과를 사용자에게 표시
3. 사용자가 결과를 검증/수정
4. 검증된 결과를 라벨로 확정
5. 확정된 라벨 데이터를 Subset에 추가
6. 보강된 데이터로 모델 재학습
7. 반복
```

### 가치
- 초기에는 적은 라벨 데이터로 시작
- 추론 → 검증 → 재학습 사이클을 반복하며 점진적으로 모델 품질 향상
- Active Learning 패러다임의 실질적 구현

## 구현 시 고려사항

1. **파이프라인 검증**: 순환 참조(cycle) 감지, 타입 호환성 검사
2. **실행 엔진**: DAG 토폴로지 정렬 → 의존성 순서대로 실행
3. **에러 핸들링**: 중간 노드 실패 시 전체 파이프라인 중단/부분 실행 정책
4. **리소스 관리**: 동시 실행 파이프라인 수 제한, 큐잉
5. **결과 저장**: 파이프라인 실행 히스토리 및 중간 결과 저장
6. **파이프라인 템플릿**: 자주 사용하는 패턴을 템플릿으로 제공
