# Vision Flow - Core Vision

## 프로젝트 정의

Vision Flow는 **웹 기반 Vision AI 모델 학습/추론 파이프라인 관리 서비스**이다.

사용자가 이미지 데이터를 업로드하고, 라벨링하고, 모델을 학습시키고,
학습된 모델로 추론하고, 여러 모델을 연결하여 파이프라인을 구성하고,
추론 결과를 다시 학습 데이터로 활용하는 전체 사이클을 하나의 플랫폼에서 관리한다.

## 핵심 가치 제안

### 경쟁 서비스와의 차별점

기존 Vision AI 플랫폼들 (Roboflow, V7 Labs 등)은 다음의 기본 흐름을 제공한다:
```
업로드 → 학습 → 평가 → 추론
```

Vision Flow는 여기에 두 가지 핵심 차별점을 더한다:

#### 1. 파이프라인 오케스트레이션
- **노드 기반 시각적 편집기** (React Flow)로 여러 모델을 연결
- 모델 A의 출력 → 전처리 → 모델 B의 입력 → 후처리 → 최종 결과
- 드래그앤드롭으로 복잡한 추론 파이프라인을 시각적으로 구성
- 예시: 물체 감지 → 크롭 → 분류 → 결과 취합

#### 2. 데이터 피드백 루프
- 추론 결과를 학습 데이터로 재활용
- 모델이 추론한 결과에 대해 사람이 검증/수정 → 새 학습 데이터로 편입
- 지속적인 모델 개선 사이클 구축
- Active Learning 패러다임의 실질적 구현

## 타겟 사용자

### 1차 타겟
- **Vision AI 엔지니어**: 모델 학습/배포 파이프라인을 효율적으로 관리하고 싶은 엔지니어
- **ML 팀**: 데이터 관리, 모델 버전 관리, 파이프라인 구성을 통합 도구로 진행하고 싶은 팀

### 2차 타겟
- **비전공 사용자**: 코드 없이 Vision AI를 활용하고 싶은 도메인 전문가
- 파이프라인 에디터의 직관적 UI로 프로그래밍 없이 모델 조합 가능

## 제품 구성 요소

### 1. 데이터 관리
- **Data Pool**: 원본 이미지 저장소 (자유 추가/삭제, 폴더 정리)
- **Subset**: 목적별 이미지 큐레이션 (Task Type 지정, 버전 관리)
- 하나의 이미지가 여러 Subset에 속할 수 있음

### 2. 라벨링
- **자체 라벨링 도구**: Konva.js 기반 (Label Studio 미사용 — D003)
- **지원 작업**: Bounding Box, Polygon, Keypoint, Classification
- **AI 보조 라벨링**: SAM, Grounding DINO 통합 (예정)

### 3. 학습
- **지원 모델**: YOLO 시리즈 (v11 nano ~ extra-large)
- **비동기 학습**: Celery를 통한 백그라운드 학습
- **실시간 모니터링**: SSE로 학습 메트릭 스트리밍 (예정)
- **모델 버전 관리**: 시맨틱 버저닝 vX.Y.Z (D016)

### 4. 추론
- **단일 모델 추론**: 이미지 → 모델 → 결과
- **파이프라인 추론**: 여러 모델/전처리/후처리를 연결

### 5. 파이프라인
- **노드 기반 편집기**: React Flow 활용
- **노드 유형**: 입력, 전처리, 모델, 후처리, 출력, 분기, 합류
- **실행 관리**: 파이프라인 실행/중지/모니터링

## AI 라이브러리 선택 근거

### Ultralytics (YOLO)
- Vision AI에서 가장 널리 사용되는 실시간 객체 탐지 프레임워크
- 단일 API로 학습/추론/내보내기 지원
- Classification, Detection, Segmentation, Pose Estimation 모두 지원
- 커뮤니티 활발, 문서화 충실

### SAM (Segment Anything Model) — 예정
- Meta에서 공개한 범용 세그멘테이션 모델
- 한 번의 클릭/박스 지정으로 객체 마스크 자동 생성
- 라벨링 도구에서 AI 보조 기능으로 활용

### Grounding DINO — 예정
- 텍스트 프롬프트로 객체를 자동 감지
- "자동차"라고 입력하면 이미지에서 모든 자동차 위치를 찾아줌
- Zero-shot 객체 탐지로 라벨링 효율 극대화
